# Assess command for vibe-coded repositories

Ghostbench will make `assess <repoPath>` the primary CLI shape for the vibe-coded repository pivot, with app briefs supplied by CLI text, brief files, or case files. The old `run <casePath>` and `compare <casePath>` commands came from agent-response evaluation, but app owners think in terms of assessing a repository's readiness rather than running a benchmark case.
